Cartly Documentation

Welcome to Cartly – Smart Conversion Cart for WooCommerce. This guide covers installation, all features, and troubleshooting.

✅ Zero configuration needed. Cartly works automatically. After activation, visit any product page — the cart is already live. No shortcodes or template edits required.

Installation

Method 1 — WordPress Dashboard (Recommended)

1
Upload the ZIP Go to WordPress Admin → Plugins → Add New → Upload Plugin. Select cartly.zip and click Install Now.
2
Activate Click Activate Plugin. You will be redirected to the Cartly settings page automatically.
3
Done! The cart is now live on your store. Visit any product page to see it.

Method 2 — FTP / cPanel

  1. Unzip cartly.zip on your computer
  2. Upload the cartly/ folder to /wp-content/plugins/
  3. Go to WordPress Admin → Plugins → Find "Cartly" → Activate

Quick Start (2 Minutes)

These 4 steps are all you need to go from installed to conversion-optimized:

1
Choose a Style Preset Go to Cartly → Settings → left panel. Click one of the 6 presets (Modern, Minimal, Dark, Glass, Gold, Bloom). Colors update instantly.
2
Set Free Shipping Threshold Cart Drawer tab → Free Shipping Bar → set the amount to match your WooCommerce free shipping rule (e.g. $50).
3
Enable Upsells Upsells tab → toggle ON → set strategy to "Related Products". Products appear automatically.
4
Save & Preview Click Save Settings → click Preview (top right) → add a product to cart.
💡 Tip: The Sticky Add-to-Cart Bar (Sticky tab) is enabled by default. Visit a product page and scroll down to see it appear.

Cart Drawer

The cart drawer slides in from the right (or left) when a customer adds a product, clicks the floating button, or a trigger fires. It works automatically on all pages — no template edits needed.

Floating Button

The floating cart button is always visible in the corner. It shows a live item count badge that updates without page reload. On mobile, it can be positioned higher to avoid overlapping theme navigation bars.

Free Shipping Bar

Shows customers exactly how much more they need to spend to unlock free shipping. Set the threshold to match your WooCommerce → Shipping → Free Shipping setting. The bar turns green and shows a success message when the goal is reached.

Mobile Bottom Sheet

On screens under 600px wide, the drawer automatically transforms into a bottom sheet that slides up from the bottom — matching the UX of native mobile apps. No configuration needed. Customers can swipe down to close it.

Sticky Add-to-Cart Bar Unique Feature

This is Cartly's headline feature — no other WooCommerce cart drawer plugin includes this.

On product pages only, a sticky bar appears at the bottom of the screen as soon as the native WooCommerce Add to Cart button scrolls above the viewport. It shows:

  • Product thumbnail
  • Product name and price
  • Quantity selector (optional)
  • Add to Cart button

Clicking the button adds the product to cart and immediately opens the Cartly drawer. Variable products show a "Select Options" button that smoothly scrolls back to the native form.

💡 How it detects scroll position: It watches the native WooCommerce form.cart element. When that element's bottom edge scrolls 60px above the viewport, the bar appears. This works with all standard WooCommerce themes.

Smart Upsell Engine

Upsells appear inside the cart as horizontally scrollable product cards. Customers can add them with one tap — no page navigation.

Strategies

Same CategoryOther products from the same category as cart itemsCategory-focused stores

Rule-Based Upsells

When enabled, specific products appear when the cart total reaches your minimum. Example: "Cart > $50 → show premium accessory pack." This overrides the standard strategy above.

Enter product IDs separated by commas. To find a product ID: go to WooCommerce → Products, hover a product — the ID appears in the URL at the bottom of your browser.

Frequently Bought Together

Changes the upsell section title and uses your products' cross-sell items as suggestions, mimicking the Amazon "Frequently Bought Together" pattern.

Cart Goals & Rewards System

Multi-Tier Goals Bar

Shows a visual progress track with 3 milestone icons. Icons are greyed out until each amount is reached, then animate into color. A message below tells customers how much more they need for the next reward.

Example setup:

  • Goal 1: $50 → 🚚 Free Shipping
  • Goal 2: $80 → 🎁 Free Gift
  • Goal 3: $120 → 💎 10% Discount
⚠️ Important: The goals bar only displays the milestones visually. You must set up the actual discounts/shipping rules separately in WooCommerce.

Exit Intent

Detects when the mouse cursor leaves the browser viewport toward the address bar — a reliable signal that the user is about to leave. The floating cart button pulses, then the cart opens after a brief delay.

Desktop Only Mobile devices have no cursor, so exit intent has no effect on phones/tablets.

AJAX Actions

All endpoints are registered for both logged-in users (wp_ajax_) and guests (wp_ajax_nopriv_). Every request requires a valid cartly_nonce nonce.

ActionPOST ParametersDescription
cartly_get_cart—Returns full cart contents as JSON.
cartly_add_to_cartproduct_id, qty, variation_idAdds a product to the cart.
cartly_update_cartkey, qtyUpdates item quantity. qty=0 removes the item.
cartly_remove_itemkeyRemoves a cart item by its key.
cartly_apply_couponcouponApplies a coupon code to the cart.
cartly_remove_couponcouponRemoves an applied coupon from the cart.
cartly_get_upsells—Returns upsell product data for the current cart.
Developer Note: Cartly does not currently expose apply_filters() hooks for modifying rendered output. Use the standard WooCommerce and WordPress hooks above to extend behaviour, or open a support ticket to request a specific filter.

Frequently Asked Questions

The cart doesn't appear on my site. What's wrong?

Check: (1) WooCommerce is installed and active. (2) The master enable toggle at the top of Cartly settings is ON. (3) You are not on the checkout page (Cartly is intentionally excluded there). See also Troubleshooting.

Do I need to edit any theme templates?

No. Cartly works entirely through standard WordPress hooks. The cart renders in the wp_footer action — no template edits needed.

The sticky add-to-cart bar doesn't appear.

The bar only appears on single product pages (WooCommerce is_product()). Scroll down on a product page until the Add to Cart button is above the screen — the bar slides up then.

Can I use Cartly on non-WooCommerce pages?

Yes. By default, Cartly loads on all pages. Enable "Load Only on WooCommerce Pages" in the Advanced tab if you want to restrict it.

Will Cartly conflict with my checkout plugin?

Cartly is excluded from is_checkout() pages automatically. It should not conflict with checkout plugins.

How do I set up the free shipping bar?

Set the threshold in Cartly → Cart Drawer → Free Shipping Threshold. This must match the minimum order value in WooCommerce → Shipping → Shipping Zones → Free Shipping method.

Can customers apply coupons inside the cart?

Yes. Enable Coupon Field in the Cart Drawer tab. The field appears in the cart footer and supports all WooCommerce coupon types.

Troubleshooting

Cart loads but updating quantity doesn't work

  1. Check browser console for JavaScript errors
  2. Disable other cart plugins temporarily (conflicts can happen with multiple cart plugins)
  3. Ensure WooCommerce cart fragments script is loading (wc-cart-fragments)

Style presets don't apply colors to the frontend

  1. Click Save Settings after selecting a preset
  2. Clear your site cache (if using WP Rocket, W3 Total Cache, etc.)
  3. Check that your theme is not overriding the CSS custom properties with !important

Admin settings save but nothing changes on frontend

  1. Clear browser cache (Ctrl+Shift+R)
  2. Clear server-side cache (caching plugin, CDN, or hosting cache)
  3. Check the inline <style> tag in your page source — it should contain --cly-pri variables

Getting a PHP error after activation

  1. Ensure WooCommerce is installed and active before activating Cartly
  2. Check PHP version meets the 7.4 minimum requirement
  3. Enable WP_DEBUG and check the debug log at wp-content/debug.log

Sticky ATC bar not visible on product pages

  1. Confirm Sticky ATC is enabled in Cartly → Sticky tab
  2. The bar only appears when you scroll down past the native Add to Cart button
  3. Some page builders rename the form class from form.cart — contact support if this applies
